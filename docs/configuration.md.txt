Configuration Guide
This document explains how to configure the Outlook Signature Intelligence Extraction Engine for different environments.
________________________________________
Folder Selection
The file:
/config/folders.txt
contains a list of Outlook folder paths to scan.
Example (anonymized):
Inbox\2025\03_March
Inbox\2025\04_April
Inbox\2025\05_May\Client_A
Each line represents a folder root.
Phase 1 allows scanning:

________________________________________
Ignored Senders
The file:
/config/ignored_senders.csv
contains a list of email addresses that should be excluded from processing.
Example:
no-reply@example.com
notification@example.com
support@example.com
These addresses often belong to automated systems.
________________________________________
Keyword Dictionaries
/config/job_title_keywords.txt
Used to identify job titles such as:
manager

lead
head of
/config/department_keywords.txt
Used to classify departments:
marketing

/config/extra_title_tokens.txt
Additional variants of job title terms.
/config/extra_department_tokens.txt
Additional department phrasing or synonyms.
________________________________________
Notification Detection
The file:
/config/notification_patterns.txt
contains patterns used to exclude automated emails (e.g., GitHub notifications, ticketing systems).
________________________________________
Meta File List
A helper file:
/config/meta_files.txt
lists cache files available for Phase 2 processing.
